TANDEM-DIMPLE Webserver Tutorial

This webserver supports two core workflows: Model Inferencing and Transfer Learning. Follow the steps below to create sessions, submit jobs, and review results.

Core functionalities

  1. Model Inferencing: Predict the pathogenicity of single amino acid variants (SAVs) using pre-trained models.
  2. Transfer Learning: Train and fine-tune gene/disease-specific models using curated variant datasets to improve prediction accuracy.

Quick definitions

  • Session ID: Similar to a user ID. One session can contain multiple jobs.
  • Job: A single run (Inference or Transfer Learning) under a Session ID.
  • SAV format: <UniProt_ID> <Mutation> (and optional label for training).

Step-by-step guide

  1. Step 1: Create a new session or resume a previous one

    Enter a Session ID to resume an existing session, or leave the input empty to generate a new Session ID. A Session ID is similar to your user ID; under one Session ID, you can have multiple job IDs.

  2. Step 2: Configure and submit a job
    1. Select Task Type: Choose Inference or Transfer Learning.
    2. Provide SAVs (paste into the textbox or upload a .txt file)

      Inference format:

      P29033 Y217E
P29033 Y217F
P29033 Y217T

      Transfer Learning format (include label):

      P29033 Y217E 1
P29033 Y217F 0
P29033 Y217T 1
      Notes:
      • SAVs must be separated by new lines.
      • If both text input and file upload are provided, the server will use the uploaded file.
      • Labels: 1 = pathogenic, 0 = benign.
    3. Provide a protein structure ID or upload a coordinate file (Optional)
      • PDB ID: retrieved from the Protein Data Bank.
      • UniProt ID: an AlphaFold2 predicted structure is used if available.

      If no structure is provided, the system automatically selects a representative structure from the UniProt ID using: (1) sequence coverage, (2) structural resolution, and (3) PDB deposition date.

    4. Select a model (Inference Mode only)

      Available models include:

      • TANDEM (gene/disease-general model)
      • TANDEM-DIMPLE for GJB2 (gene/disease-specific model)
      • TANDEM-DIMPLE for RYR1 (gene/disease-specific model)

      User-trained models from previous sessions may also appear in the dropdown menu.

    5. Submit the job

      Click Submit to start processing.

  3. Step 3: Monitor job status and analyze results
    1. Submission Status displays all submitted parameters or reports errors.
    2. Job Select displays old jobs under the current session.
    3. The first window of Result Output shows real-time execution.
    4. Pathogenicity probability is reported for each SAV, classified as:
      • Pathogenic (probability ≥ 0.5)
      • Benign (probability < 0.5)
    5. Download all raw output files by clicking the ↓ button on the right-most side of the results.
