Q1: What is a single amino acid variant (SAV)?

A SAV refers to a change in a protein's primary sequence caused by a nucleotide substitution that alters the encoded amino acid. TANDEM website records each SAV using the UniProt ID followed by the amino acid substitution (e.g., P29033 Y217D). To identify the correct UniProt ID for a gene or protein, please refer to UniProt.

Q2: What is the prediction of protein pathogenicity?

The protein pathogenicity prediction gives the probability of a specific SAV is disease-causing (pathogenic) or harmless (benign) by considering its effects on protein sequence, structure, dynamics, and biological function.

These probabilities can be generated using either TANDEM (gene/disease-general model) or TANDEM-DIMPLE (gene/disease-specific models), trained on corresponding general or curated gene/disease-specific datasets.

Q3: What is transfer learning?

Transfer learning is a machine-learning strategy in which a model pretrained on a large, general dataset is further fine-tuned using a smaller, gene/disease-specific dataset. This approach leverages previously learned knowledge to improve prediction accuracy for specific genes or diseases, especially when labeled data are limited.

Q4: What is SHAP analysis?

A SHapley Additive exPlanations (SHAP) value measures a feature's contribution to a model's prediction. It is derived from the Shapley value in cooperative game theory, which defines how to distribute a game's payout among players such that each player's payout is proportional to that player's contribution [1]. In our setting, the game's payout is the model output for a given input SAV; the players are protein features associated with that SAV; and each player's payout corresponds to the contribution of that feature to the predicted output.

Formally, consider a predictive model f: ℝn → ℝ evaluated at a specific input x = (x1, ..., xn), and let N = {1, ..., n} index the xi features. The marginal contribution of feature i to S, a subset of other features which does not contain i (S ⊆ N \ {i}), is defined as

Δ(i, S) = f(xS∪{i}) - f(xS)
Equation 1

, where xS denotes the input that keeps the features in S the same as in x while all other non-S features are replaced by reference values or dataset averages. S ∪ {i} means the set including the features in S and feature i. Intuitively, Δ(i, S) quantifies how much feature i adds to the model's output when it is "switched on" (present) in the context of the other features in S. The Shapley value for feature i is then defined as the weighted average of the marginal contributions of feature i over all possible subsets S ⊆ N \ {i}, given by

ϕi = ΣS⊆N\{i} αSΔ(i, S)
Equation 2

, where αS = |S|!(n−|S|−1)! / n! denotes the weight of each term, corresponding to the fraction of permutations of N in which i appears after exactly the elements of S. To explain individual predictions, SHAP then formulates the explanation model g as a linear function of binary variables, with coefficients equal to the Shapley values (or SHAP values)

g(x') = ϕ0 + Σni=1 ϕix'i
Equation 3

, where x' ∈ {0,1}n is a binary indicator vector denoting which features are present and ϕ0 = f(x∅) is the model's output with no features present. For specific input x, where all features are present (i.e., x'i = 1 ∀i), the explanation model equals the true model output

f(x) = ϕ0 + Σni=1 ϕi
Equation 4

This additive decomposition of f(x) is the essence of a SHAP explanation: each ϕi quantifies how much feature i pushed the prediction up or down relative to the baseline ϕ0. To examine global feature contribution across M data points, we average the absolute SHAP values per feature

Ii = 1/M ΣMj=1 | ϕi(j) |
Equation 5

, where ϕi(j) is the SHAP value of feature i for input j. A larger Ii indicates that feature i contributes more strongly to the model's predictions. In our work, we use the KernelSHAP method as implemented in the shap python package (version 0.41.0) to approximate these SHAP values.
