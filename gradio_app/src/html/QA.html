What is TANDEM-DIMPLE?

TANDEM-DIMPLE is a pathogenicity prediction webserver developed by the Yang Lab at National Tsinghua University, Taiwan.

Inspired by AlphaMissense, leveraging structural embeddings (Cheng et al., 2023), and Rhapsody web server, incorporating protein dynamics (Ponzoni et al., 2020), we integrated structural and dynamics-based features alongside sequence conservation signals to train our gene/disease-general DNN classifier for SAV pathogenicity prediction. This gene-general model (hereafter “TANDEM”) is sufficiently deep to achieve high predictive accuracy, outperforming AlphaMissense by 6% on a test set of more than 2,000 SAVs from the Rhapsody dataset, yet lightweight enough (<5,000 parameters) to support gene/disease-specific transfer learning (the resulting model is termed “TANDEM-DIMPLE). Specially, after transfer learning on clinically labeled SAVs for two genes, RYR1 and GJB2, whose variants cause malignant hyperthermia and hearing loss, respectively, our model achieves 98.7% accuracy for GJB2 and 97.0% for RYR1, surpassing AlphaMissense's 80.0% and 66.7%, respectively. With these efforts, our web server now serves SAV pathogenicity inferencing for general disease as well as for specific genes/diseases using user-uploaded clinically labeled SAVs, where the gene-specific models are automatically trained in our website for private or public uses.

What is a single amino acid variant (SAV)?

A SAV refers to a change in a protein’s primary sequence caused by a nucleotide substitution that alters the encoded amino acid. Although most SAVs are harmless to the host, some are so deleterious that they cause embryonic or early lethality, making such variants absent from the human genome, while a small subset impairs protein function just enough to cause diseases without preventing survival to reproductive age. TANDEM website records each SAV using the UniProt ID followed by the amino acid substitution (e.g., P29033 Y217D). To identify the correct UniProt ID for a gene or protein, please refer to UniProt.

How does TANDEM-DIMPLE Website work?

The website is deployed as a five-component pipeline, frontend, database, dispatcher, feature generation, and inference/transfer-learning, packaged as containers and run together with Docker Compose (https://www.docker.com). Compose-managed containers simplify deployment and scaling (e.g., running multiple workers) while keeping the application stack reproducible. The frontend is implemented in Python using the Gradio package (Abid et al., 2019), which builds the website interface to receive user requests and render either inferencing outputs or transfer learning results. Upon submission, the frontend sends a job request to MongoDB using PyMongo driver, which communicates over MongoDB’s native wire protocol (https://www.mongodb.com), and the new job is recorded with status “pending”. A background dispatcher then claims the pending job with earliest creation times from the database using a find-and-update operation in PyMongo and updates its status to “processing”. The dispatcher serializes the job payload and forwards it via HTTP to next container, which generates 33 features in the TANDEM feature set, and then to the inference/transfer-learning module. How TANDEM-DIMPLE model was trained?

TANDEM models were trained using deep neural networks (DNNs) on the R20000 set with the TANDEM feature set (?). The input features were standardized prior to training. The network comprises 5 hidden layers with GeLU activation: four layers have 33 neurons, and the last layer has 10, which is connected to the two-neuron output layer that uses the softmax function to report the pathogenicity probabilities for a SAV. Training used a mini-batch size of 300, Xavier uniform initialization, the Nadam optimizer, and cross-entropy loss with L2 regularization (1e-4). The training was stopped when the validation loss does not get lower after 50 consecutive epochs. The model had 4,850 trainable parameters and was implemented in TensorFlow v2.17.0. The trained weights from 5 TANDEM models were then used to initialize gene-specific training on GJB2knw and RYR1knw datasets, producing 15 TANDEM-DIMPLE models for GJB2 and 15 for RYR1, respectively. The gene-specific models used the same architecture and hyperparameters as described above.

Taken from the Rhapsody (Ponzoni et al., 2020), the 20,361 clinically annotated SAVs of 2,423 genes comprises 13,626 pathogenic and 6,735 benign variants, referred as R20000 set. The dataset integrates information from five publicly available datasets with refined annotations and pathogenic/benign labels using three well-known databases UniProtKB (UniProt, 2023), ClinVar (Landrum et al., 2020), and HuVarBase (Ganesan et al., 2019). All SAVs have the corresponding UniProt IDs, including a total of 2,423 UniProt protein sequences, each of which can harbor one or multiple SAVs.

Performance was evaluated by averaging over 5 TANDEM or 15 TANDEM-DIMPLE models using 5 metrics: Accuracy, Area Under Curve Receiver Operating Characteristic (AUC ROC or AUC), Precision, Recall, and F1.

We coined TANDEM as gene/disease-general model and TANDEM-DIMPLE as gene/disease-specific model.

What is SHAP analysis?

A SHapley Additive exPlanations (SHAP) value measures a feature's contribution to a model's prediction. It is derived from the Shapley value in cooperative game theory, which defines how to distribute a game's payout among players such that each player's payout is proportional to that player's contribution (Lundberg & Lee, 2017). In our setting, the game's payout is the model output for a given input SAV; the players are protein features associated with that SAV; and each player's payout corresponds to the contribution of that feature to the predicted output.

Formally, consider a predictive model f: ℝn → ℝ evaluated at a specific input x = (x1, ..., xn), and let N = {1, ..., n} index the xi features. The marginal contribution of feature i to S, a subset of other features which does not contain i (S ⊆ N \ {i}), is defined as

Δ(i, S) = f(xS∪{i}) - f(xS)
Equation 1

, where xS denotes the input that keeps the features in S the same as in x while all other non-S features are replaced by reference values or dataset averages. S ∪ {i} means the set including the features in S and feature i. Intuitively, Δ(i, S) quantifies how much feature i adds to the model's output when it is "switched on" (present) in the context of the other features in S. The Shapley value for feature i is then defined as the weighted average of the marginal contributions of feature i over all possible subsets S ⊆ N \ {i}, given by

ϕi = ΣS⊆N\{i} αSΔ(i, S)
Equation 2

, where αS = |S|!(n−|S|−1)! / n! denotes the weight of each term, corresponding to the fraction of permutations of N in which i appears after exactly the elements of S. To explain individual predictions, SHAP then formulates the explanation model g as a linear function of binary variables, with coefficients equal to the Shapley values (or SHAP values)

g(x') = ϕ0 + Σni=1 ϕix'i
Equation 3

, where x' ∈ {0,1}n is a binary indicator vector denoting which features are present and ϕ0 = f(x∅) is the model's output with no features present. For specific input x, where all features are present (i.e., x'i = 1 ∀i), the explanation model equals the true model output

f(x) = ϕ0 + Σni=1 ϕi
Equation 4

This additive decomposition of f(x) is the essence of a SHAP explanation: each ϕi quantifies how much feature i pushed the prediction up or down relative to the baseline ϕ0. To examine global feature contribution across M data points, we average the absolute SHAP values per feature

Ii = 1/M ΣMj=1 | ϕi(j) |
Equation 5

, where ϕi(j) is the SHAP value of feature i for input j. A larger Ii indicates that feature i contributes more strongly to the model's predictions. In our work, we use the KernelSHAP method as implemented in the shap python package (version 0.41.0) to approximate these SHAP values.

What features are we using in the TANDEM-DIMPLE models?

For each SAV, we derived 33 protein features, grouped into three categories: 9 sequence chemical features, 15 structure features, and 9 dynamics features. Among these, 7 features were identical to the ones used in the Rhapsody study (Ponzoni et al., 2020). Dynamics features were computed using Gaussian Network Models (GNM) (Bahar et al., 1997; Yang et al., 2006) to capture the residue fluctuations and global motion of protein structures. For proteins that have interaction partners such as membrane proteins or protein-nucleic acid complexes, we used envGNM (Li et al., 2017; Li et al., 2020), which models proteins' surrounding contacts as the “environment” that includes lipid membrane, other oligomers or nucleic acids/ large ligands. Structure features describe the global and local topology of the protein in 3D space such as radius of gyration or distance to the center of mass, secondary structures, intrinsic disorder regions, and solvent-accessible surface area, along with local interactions such as residue contacts, hydrogen bonds, and disulfide bonds were also computed. Sequence and chemical features included residue-level physicochemical properties such as polarity, charge, residue conservation that was assessed using ConSurf (Ben Chorin et al., 2020; Goldenberg et al., 2009), BLOSUM62 (Henikoff & Henikoff, 1992), and PSIC (Sunyaev et al., 1999). Both the ConSurf score of the SAV site of interest and that of its spatial neighbors in 3D were calculated.
